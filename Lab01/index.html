Sobre el ciclo de vida y desarrollo de los sistemas de información:
¿Cuál es el ciclo de vida de los sistemas de información?
¿Cuál es el ciclo de desarrollo de sistemas de información?

El ciclo de vida de un sistema de información es un enfoque por fases del análisis y diseño que sostiene que los sistemas son desarrollados de mejor manera mediante el uso de un ciclo especifico de actividades del analista y del usuario
Fases:
Nacimiento
Surge de una necesidad o requerimiento por parte del usuario, ver si justifica el desarrollo del sistema, estudio de factibilidad
Desarrollo
Analizan requerimientos y se elabora un diseño, se elaboran los programas necesarios para que el sistema pueda operar.
Diseñar, construir, y/o adecuar los programas que se requieren
Operacion
El sistema ya esta terminado, el usuario trabaja en el mismo, si el sistema no satisface los requerimientos o hay algun error es necesario pasar a la fase de mantenimiento
Mantenimiento
Corregir los errores que se detectan en el programa o en las funciones. En esta fase el usuario puede agregar nuevos requerimientos
Muerte
Llega aqui cuando deja de ser necesario o cuando debe de reemplazarse por otro mejor. Si el sistema original ya ha sido modificado el proceso se inicia nuevamente, pues el programa anterior ya murio y ahora seria uno nuevo.
Fases de desarrollo de sistemas de informacion:
Planeacion
Requerimiento completo teniendo en cuenta el flujo de informacion con que se trabaja, documentar mediante diagramas de flujo, usarios,necesidades
Analisis
Asesorar,supervisar,recomendar,y modificar procesos internos
Implantacion
Construccion del nuevo sistema y el paso del sistema a produccion
Fases:
-Probar redes y base de datos
-Construccion y prueba de las aplicaciones
-Instalacion y prueba del nuevo sistema
-Entrega del sistema para puesta en funcionamiento
Pruebas
Resultados del sistema. Precision, Calidad y los productos esperados. Es un proceso ciclico

Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

Sirve para indicar las instrucciones prioritarias para que no sean sobreescritas por alguna otra.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Por que no todas las imagenes cuentan con una calidad muy buena, no son escalables, puede distraer de la pagina

Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?

Es mejor usar porcentajes para que se ajuste a todas los dispositivos o tamaños de ventana, pues termina siendo algo subjetivo y no un valor fijo.

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Primero que nada es mucho mas conveniente que los estilos esten en un archivo aparte en el CSS, preferiblemente usar el menor numero de hojas CSS, una es mas que suficiente.
Las hojas de estilo deben descargarse rápidamente para que nuestro navegador aplique los estilos lo antes posible y por ello se recomienda que no ocupen más de 100 kb por fichero CSS.
